# Schedule University

**Автоматическое составление учебного расписания:** точная оптимизация (MILP) и эволюционный подход (генетический алгоритм) на одном формате входных данных. Проект ориентирован на интеграцию через HTTP API и на эксперименты с синтетическими и реальными нагрузками.

---

## Возможности

| Направление | Суть |
|-------------|------|
| **MILP** | Модель в **PuLP** (солвер CBC): жёсткие ограничения как часть модели, целевая функция штрафует мягкие нарушения. |
| **Генетический алгоритм** | Популяция живёт в **пространстве допустимых по жёстким ограничениям** решений: конструктивная инициализация, мутации с сохранением допустимости, **лёгкий групповой кроссовер** с коротким repair и откатом к родителю. |
| **API** | **FastAPI** + **Pydantic** — валидация JSON, быстрый ответ (**ORJSON**). |
| **Данные** | Один контракт: группы, предметы, преподаватели, аудитории, дни, времена, `subject_teachers`, `subject_count`. |

## Быстрый старт

### Окружение

```bash
cd MetaDiplom/src/server
pip install -r requirements.txt
```

Стек: **Python 3.10+**, **FastAPI**, **NumPy**, **PuLP**, **Pydantic**, **Uvicorn**.

### Запуск API

```bash
cd MetaDiplom/src/server
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

Проверка: откройте `http://127.0.0.1:8000/` — ответ `{"result":"Hello"}`.

## API

### MILP — `POST /solve_pulp`

Тело: объект **InputData** (см. ниже). Ответ: `result` — список строк расписания в формате, совместимом с `MyPulp.get_json_list()`.

### GA — `POST /solve_genetic`

Тело: **SolveRequest** = поля **InputData** + опционально `ga_params` (см. `schemas.GAParams`).

Ответ:

```json
{
  "result": [["group", "day", "time", "subject", "room", "teacher"], ...],
  "method": "ga",
  "meta": {
    "penalty": <int>,
    "history": [<лучший штраф по поколениям>]
  }
}
```

`penalty` — сумма **жёсткого** и **мягкого** штрафа; при успешной оптимизации мягких критериев может стремиться к **0** (жёсткие при корректной постановке и данных остаются нулевыми).

Пример вызова (файл из `tests/`):

```bash
curl -s -X POST http://127.0.0.1:8000/solve_genetic ^
  -H "Content-Type: application/json" ^
  -d @MetaDiplom/tests/input_30.json
```

> В `main.py` гиперпараметры GA берутся из `GAParams` по умолчанию; при необходимости расширьте тело запроса полем `ga_params` (см. `schemas.py`).

## Формат входных данных (кратко)

| Поле | Назначение |
|------|------------|
| `groups`, `subjects`, `teachers`, `rooms`, `days`, `times` | Списки сущностей (порядок задаёт индексы в хромосоме GA). |
| `subject_teachers` | Для каждого предмета — список допустимых преподавателей. |
| `subject_count` | Для группы — сколько пар по предмету; для отсутствующих пар используется `default_count`. |
| `default_count` | Значение по умолчанию для пар группа–предмет, не перечисленных в `subject_count`. |

**Разрешимость:** число занятий у одной группы не должно превышать `|days| × |times|`. Для предметов с **одним** преподавателем суммарное число пар по этому предмету не может быть больше числа временных слотов в неделю (один слот — не более одной пары у данного преподавателя). Иначе **ни MILP, ни GA** не смогут выдать корректное расписание.

## Генетический алгоритм (идея реализации)

- **Хромосома:** для каждого атомарного занятия — `(день, время, аудитория, преподаватель)`.
- **Жёсткие ограничения:** уникальность слота в группе; уникальность аудитории и преподавателя в глобальном слоте; преподаватель ∈ `subject_teachers[предмет]`.
- **Мягкие:** «окна» в расписании групп и преподавателей, штраф за избыточное число учебных дней у группы.
- **Операторы:** конструктивная инициализация; мутации (обмен в группе, смена аудитории/учителя, перенос слота); **групповой кроссовер** с ограниченным числом попыток repair и откатом к родителю; опционально локальный поиск по допустимым соседям.
